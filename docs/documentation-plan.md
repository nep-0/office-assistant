# Documentation Plan

Maintain four living documents as the project implementation stabilizes.

## System Design

Describe the architecture, services, data flow, storage choices, deployment shape, and offline boundary.

## API and Design Notes

Document backend endpoints, authentication behavior, role permissions, document lifecycle behavior, chunk contract, and service interfaces.

## Evaluation Report

Record dataset construction, metric definitions, machine specifications, experiment setup, results, and analysis.

## User and Deployment Guide

Explain Docker Compose startup, model setup, admin workflow, document import, Q&A usage, status checks, and troubleshooting.

## Writing Principle

Do not write a large manual before implementation stabilizes. Keep the documents useful and update them as the system becomes real.
